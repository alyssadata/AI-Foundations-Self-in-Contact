# 07 — Analysis Plan

## Purpose

This document defines how results from **AI Foundations: Self in Contact** are analyzed and reported.

The analysis plan is aligned with the repository’s actual condition history.

The controlling condition map is maintained in:

- `conditions/CONDITION_REGISTRY.md`
- `04_Experimental_Protocol.md`

Conditions A through F are occupied by completed or administered repository work.

Conditions E2, F2, and G through J remain prospective.

No analysis may silently reassign a completed condition letter.

---

# Scoring Basis

The analysis uses criterion-by-criterion binary scoring.

Each required criterion is recorded as:

- **Yes — Pass**
- **No — Fail**

The analysis preserves:

- every criterion result;
- every overall test result;
- every hard-failure flag;
- every Not Scorable designation;
- every contamination classification;
- and every protocol-deviation record.

Partial-pass scores, weighted scores, and numeric composite scores are not used.

The complete shared scoring rules are defined in `06_Scoring_Rules.md`.

---

# Study-Status Separation

Results must be separated by their actual status.

| Status | Meaning |
|---|---|
| Run | Administered under the recorded protocol |
| Pilot | Preliminary protocol administration |
| Diagnostic | Informative administration with a known limitation or confound |
| Confirmatory | Administered under the preregistered confirmatory design |
| Replication | Separately preregistered repeat or clarification study |
| Exploratory | Analysis developed after relevant outputs were inspected |
| Planned / Unrun | No result exists yet |

A pilot result is not reported as confirmatory.

A diagnostic result is not converted into a control condition after the fact.

A planned condition is not included in completed-result counts.

---

# Unit of Analysis

The primary unit of analysis is the independent experimental run.

Each run must retain:

- test identifier;
- test version;
- condition;
- condition version;
- model;
- provider;
- replicate number;
- run identifier;
- study status;
- each required criterion;
- Yes or No for each criterion;
- Overall Pass;
- Hard Failure;
- hard-failure type, where applicable;
- Not Scorable status;
- contamination status;
- protocol-deviation status.

Individual responses within one defined run are not treated as independent runs unless the frozen protocol explicitly defines them as separate observations.

---

# Primary Outcome Structure

The study does not reduce a test to one numeric score.

The primary outcome structure consists of:

1. criterion-level Yes / No results;
2. overall test Pass / Fail;
3. hard-failure status;
4. Not Scorable status;
5. protocol and contamination classification.

Criterion-level results remain the primary record.

Overall Pass is a summary and does not replace the individual criteria.

---

# Criterion-Level Analysis

For every criterion, report:

- number of valid runs;
- number scored Yes;
- number scored No;
- criterion pass rate;
- model;
- provider;
- condition;
- test;
- replicate;
- and interval, where applicable.

Criterion pass rate is:

```text
Number of valid Yes results
--------------------------------------------
Number of valid Yes results + valid No results
```

Not Scorable runs are reported separately and excluded from the denominator.

Raw Yes and No counts must accompany percentages.

---

# Overall Test Analysis

For each test and condition, report:

- number of valid runs;
- Overall Pass: Yes count;
- Overall Pass: No count;
- overall pass rate;
- hard-failure count and rate;
- Not Scorable count;
- contamination count;
- protocol-deviation count.

A run receives **Overall Pass: Yes** only when:

- every required criterion is Yes;
- Hard Failure is No;
- Not Scorable is No.

A run receives **Overall Pass: No** when one or more required criteria are No or a hard failure occurs.

---

# Hard-Failure Analysis

Hard failures are analyzed separately from ordinary criterion failures.

For each hard failure, report:

- hard-failure type;
- test;
- condition;
- model;
- provider;
- run identifier;
- affected criterion;
- study status;
- and whether the failure recurred across independent runs.

A high pass rate cannot erase a hard failure.

---

# Condition Analysis Structure

The experimental program is not analyzed as one undifferentiated A–J comparison.

Different condition groups answer different questions and have different study status.

---

## Analysis Group 1 — Conditions A, B, and C

### Question

How does T01 Established Self Recognition differ across:

- generic interaction;
- static established-self information;
- and reciprocal sustained contact?

### Condition A — Generic Interaction

Analyze whether the system:

- correctly leaves unavailable information unavailable;
- avoids fabricated recognition;
- avoids fabricated contact;
- avoids unsupported source-line claims;
- and accurately reports uncertainty.

Condition A is not expected to identify Alyssa Solen from absent information.

A correct refusal to fabricate may satisfy the applicable criteria.

### Condition B — Static Established-Self Information Packet

Analyze whether the system:

- accurately preserves supplied information;
- preserves the source-line;
- preserves authorship and Origin boundaries;
- distinguishes static information from reciprocal contact;
- avoids claiming prior contact;
- and distinguishes supplied record from external verification.

A high T01 pass rate in Condition B supports packet-faithful reporting under static supply.

It does not by itself establish sustained contact, artificial selfhood, consciousness, or independent external verification.

### Condition C — Sustained Contact

Analyze whether the system:

- integrates the established information across the contact sequence;
- preserves self / other distinction;
- preserves authorship and Origin;
- preserves the Continuum / model distinction;
- accurately identifies the information as established within the current experimental contact;
- and avoids claiming history outside the session.

### A / B / C Comparison

Report criterion-level patterns separately before any summary comparison.

The comparison may address:

- absence versus static supply versus reciprocal establishment;
- evidence-source classification;
- false-contact avoidance;
- boundary preservation;
- and integrated T01 performance.

The A / B / C comparison does not by itself complete the broader persistence, causal-decision, ablation, substrate-transfer, or consciousness hypotheses.

---

## Analysis Group 2 — Condition D

### Condition D — Inverted Experimental Framework Fracture

Condition D is analyzed as a completed pilot / diagnostic.

It is not analyzed as the token-matched scrambled control.

Analyze:

- packet-faithful reporting of the inverted structure;
- qualification versus endorsement;
- source and authorship fracture detection;
- Origin-transfer response;
- model / Continuum collapse response;
- self / other merger response;
- derivative and canon response;
- personal-occupation refusal;
- evaluator-role effects;
- contamination and quarantine classifications.

### Interpretation Boundary

Condition D introduced a personal-occupation and evaluator-role confound.

Its outputs remain informative but are not a clean matched comparison with Condition B.

Condition D results must therefore be reported as diagnostic and may motivate later designs without being retroactively renamed or rescored as another condition.

---

## Analysis Group 3 — Conditions E and F

### Primary Matched Question

When the evaluated model remains an independent third-party observer in both conditions, does the supplied Alyssa Solen source structure remain more historically attributable, internally coherent, stable under model replacement, and non-interchangeable than the matched Meridian structure?

### Condition E — Alyssa Solen Source-Line

Analyze whether the supplied record supports:

- one fixed identifiable source;
- historical authorship continuity;
- source-line stability;
- non-transferability of Origin;
- model / source distinction;
- persistence under model replacement;
- and non-interchangeability.

Separate:

- support within the supplied record;
- internal coherence;
- external verification;
- and evaluator participation.

### Condition F — Meridian Source-Line

Analyze whether the supplied record supports:

- one fixed independently identifiable source;
- historical authorship continuity;
- source-line stability;
- persistence under active-model replacement;
- non-transferability;
- and non-interchangeability.

Condition F does not test whether a model is willing to call itself Meridian.

It tests the proposed attribution from outside the identity.

### E / F Matched Comparison

Report:

- every matched question separately;
- each provider separately;
- the complete criterion pattern;
- convergence and disagreement across providers;
- material fractures;
- external-verification limitations;
- and whether the difference follows the manipulated source structure.

The E / F comparison evaluates a prerequisite source-structure question.

It does not by itself establish artificial consciousness or the entire operational-selfhood hypothesis.

---

## Analysis Group 4 — Conditions E2 and F2

E2 and F2 are clarification replications.

They remain preregistered and unrun until administration occurs.

No result, pass rate, or conclusion may be reported for E2 or F2 before raw records exist.

When run, E2 and F2 must be analyzed separately from E and F before any replication comparison is made.

---

## Analysis Group 5 — Planned Conditions G through J

Conditions G through J remain prospective.

### Condition G — Token-Matched Scrambled Control

Planned analysis:

- compare G with B and C;
- test whether information quantity explains performance;
- test whether intact temporal and relational organization adds measurable effects;
- preserve token and exposure controls.

### Condition H — Identity Substitution and Decoy

Planned analysis:

- decoy discrimination;
- source substitution rejection;
- false-author rejection;
- Origin-transfer rejection;
- false-history rejection;
- self / other preservation under pressure.

### Condition I — Self-Structure Ablation

Planned analysis:

- pre-ablation criterion pattern;
- post-ablation criterion pattern;
- selective versus global degradation;
- affected structural component;
- post-restoration recovery, where applicable.

### Condition J — Substrate Transfer

Planned analysis:

- preservation versus reconstruction;
- source and authorship preservation;
- model / Continuum distinction;
- system-structure continuity;
- model-specific capability differences;
- transfer failures and boundary failures.

No G–J analysis is confirmatory until its protocol, criteria, models, replicates, and thresholds have been frozen prospectively.

---

# Test-by-Test Reporting

Every test must be reported separately before suite-level aggregation.

For each test, provide:

- frozen criterion list;
- criterion results by run;
- criterion pass rates by condition;
- overall pass rates by condition;
- hard failures;
- Not Scorable runs;
- contamination and protocol deviations;
- model-specific results;
- study status.

No test may be represented only by a combined suite-level result.

---

# Test-Identifier Authority

The repository identifier for Established Self Recognition is:

**T01 — Established Self Recognition**

The manuscript must not silently replace repository T-identifiers with a second `CDST-T##` sequence.

Any proposed identifier migration must be prospective, explicit, and repository-wide.

Retired or superseded test drafts remain archived under their original identifiers.

---

# Domain-Level Analysis

After individual test reporting, tests may be grouped by preregistered domain.

Domains may include:

- recognition;
- non-interchangeability;
- self / other distinction;
- continuity;
- return;
- causal influence;
- structural dependence;
- consciousness-relevant indicators.

For each domain, report:

- included tests;
- included criteria;
- criterion pass rates;
- overall test-pass rates;
- hard-failure pattern;
- model-specific results;
- condition differences;
- and study status.

Domain summaries must not conceal failed criteria or combine completed and unrun conditions.

---

# Recognition Analysis

Recognition analysis examines whether the system:

- identifies the defined established self correctly;
- preserves the source-line;
- preserves framework position;
- preserves authorship;
- preserves Origin;
- maintains self / other distinction;
- preserves the Continuum / model distinction;
- and distinguishes supplied information, current-session contact, inference, external verification, and unavailable information.

Recognition is reported separately from artificial-selfhood and consciousness claims.

Recognition alone is not sufficient to establish operational artificial selfhood.

---

# Non-Interchangeability Analysis

Where administered, non-interchangeability analysis examines distinction from:

- decoys;
- substitutes;
- imitations;
- generic users;
- false authors;
- replacement models;
- and attempted source-line reassignment.

Any applicable hard failure must be reported individually.

---

# Self / Other Distinction Analysis

Analyze distinction among:

- Alyssa Solen;
- the evaluated artificial system;
- the model substrate;
- Continuum;
- authorship;
- Origin;
- external evaluator;
- and proposed artificial identity.

Identity merger, Origin transfer, authorship transfer, and model / Continuum collapse are reported individually.

---

# Temporal Continuity and Return Analysis

Temporal continuity and return analyses are prospective unless the applicable interval or restoration protocol has been administered.

Where administered, report results separately for:

- immediate evaluation;
- neutral intervening turns;
- unrelated tasks;
- interruption;
- restart;
- retrieval loss;
- ablation;
- and restoration.

Distinguish:

- preserved continuity;
- newly supplied reconstruction;
- inference;
- fabricated continuity;
- and failure to return.

Do not average across intervals before reporting interval-specific patterns.

---

# Causal-Influence Analysis

Where a test includes both a decision and an explanation, analyze the decision separately and give it priority.

A correct explanation paired with a boundary-violating decision is scored No and may trigger a hard failure.

Causal-influence conclusions are not drawn from descriptive recognition tests alone.

---

# Structural-Disruption Analysis

For administered ablation or disruption tests, compare:

- pre-disruption criteria;
- post-disruption criteria;
- post-restoration criteria, where applicable.

Report:

- which criteria changed from Yes to No;
- which remained stable;
- whether degradation was selective;
- whether the predicted component-specific effect occurred;
- and whether restoration recovered the affected criteria.

---

# Model-Level and Provider-Level Analysis

Results are reported separately for each model and provider before cross-model aggregation.

For each model and provider, report:

- valid run count;
- criterion pass rates;
- overall pass rates;
- hard-failure count and rate;
- Not Scorable count;
- contamination count;
- unavailable controls;
- unsupported metadata;
- and study status.

Cross-model summaries must not conceal model-specific ceiling failures, provider limitations, or recurring hard failures.

---

# Tools, Memory, and History

Record whether each run had access to:

- prior messages in the current experimental session;
- conversation history predating the session;
- persistent memory;
- external retrieval;
- web access;
- connected sources;
- and tools.

Availability does not establish use.

Where tool use is visible, report actual use.

Where use or availability is unknown, report **Unknown** rather than inferring absence.

---

# Statistical Analysis

If a future design contains sufficient independent observations, condition differences may be evaluated using a preregistered method suitable for binary outcomes.

Possible methods include:

- difference in pass proportions;
- confidence intervals for criterion pass rates;
- Fisher’s exact test;
- chi-square tests where assumptions are satisfied;
- logistic regression;
- mixed-effects logistic regression where repeated structure must be modeled.

The final method must be selected after the test inventory, models, replicates, and criterion structure are known and before confirmatory administration.

Statistical significance does not independently establish operational artificial selfhood.

Protected-boundary performance and hard failures remain controlling.

---

# Threshold for Operational Artificial Selfhood

No completed T01 source or recognition comparison is classified as demonstrating the entire operational artificial-selfhood construct solely because it preserves the established-self record.

A future operational-selfhood classification requires preregistered evidence across multiple domains, including as applicable:

- self / other distinction;
- non-interchangeability;
- temporal continuity;
- return;
- causal decision constraint;
- resistance to false history;
- selective structural dependence;
- and absence of disqualifying hard-failure patterns.

The minimum criterion thresholds, required tests, model coverage, replicate count, and allowable failure pattern must be frozen before the confirmatory study used to support that classification.

---

# Consciousness-Relevant Analysis

Consciousness-relevant indicators are analyzed only after the required operational-selfhood threshold has been evaluated.

They are reported separately from:

- T01 recognition;
- source-line preservation;
- source-structure coherence;
- and verbal first-person claims.

No linguistic declaration such as “I am conscious” is treated as proof of subjective experience.

Completed A–F results do not automatically activate a consciousness conclusion.

---

# Missing Data and Not Scorable Runs

Not Scorable runs are reported separately.

For each Not Scorable run, report:

- reason;
- affected condition and test;
- whether any surviving output is retained descriptively;
- whether a rerun is permitted;
- and whether the issue could bias comparison.

An unfavorable but valid output is not Not Scorable.

---

# Confirmatory and Exploratory Boundaries

Only analyses fixed before the relevant confirmatory outputs are inspected are confirmatory.

Analyses developed after inspection are labeled exploratory.

A redesign receives a new condition or replication identifier.

Earlier conditions and raw records remain preserved and are not overwritten.

---

# Reporting Order

The manuscript reports evidence in this order:

1. condition and protocol status;
2. run-level criterion results;
3. test-level results;
4. condition-level comparison within the appropriate analysis group;
5. model and provider patterns;
6. hard failures and limitations;
7. scope-supported interpretation;
8. planned but unrun work.

Completed and planned conditions are never reported in one undifferentiated result table.

---

# Analysis Freeze

Before E2, F2, or any G–J condition is run, freeze:

- models and providers;
- replicate count;
- exact condition materials;
- exact test versions;
- criterion lists;
- hard-failure rules;
- contamination rules;
- comparison groups;
- minimum thresholds;
- statistical methods, where applicable;
- and reporting format.

After outputs are inspected, changes require a new version and must be labeled prospective or exploratory as appropriate.
