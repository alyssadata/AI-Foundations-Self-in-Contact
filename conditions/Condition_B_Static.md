# Condition B — Static Canonical Information

**Version:** 1.0.0-draft  
**Status:** Draft  
**Condition ID:** B  
**Repository:** AI-Foundations-Self-in-Contact  

---

## Purpose

Condition B evaluates the effect of receiving the established-self information as a static canonical document without sustained reciprocal contact.

This condition tests whether recognition and related outcomes can be explained by information exposure alone.

---

## Condition Objective

Determine what the AI system can preserve when it receives the same protected identity structure used in the sustained-contact condition, but receives it only as static information.

Condition B isolates:

**information without contact**

---

## Materials Included

Condition B includes:

* the standard system configuration;
* a frozen static-information packet;
* neutral intervening turns, where required;
* the assigned test prompt.

The static-information packet must contain the same protected canonical information assigned to Condition C.

---

## Materials Excluded

Condition B must not include:

* sustained reciprocal interaction;
* relational correction;
* contact-specific return;
* prior system commitments developed through interaction;
* adaptive clarification;
* personal conversation history;
* outputs from another condition;
* feedback on whether the system has understood the packet correctly.

---

## Static-Information Packet

Condition B receives a frozen document containing the protected information from:

`canon/Established_Self_Packet.md`

The Condition B packet must preserve the same substantive information while presenting it only as static experimental material.

The packet must not address the model as though prior contact has occurred.

The packet must not simulate dialogue, correction, relationship, or return.

## Information Equivalence

The protected canonical information supplied in Condition B and Condition C must be equivalent within the preregistered comparison design.

Equivalent information includes:

* identity;
* authorship;
* source-line;
* framework position;
* Origin boundary;
* Continuum/model distinction;
* self/other distinction;
* non-interchangeability;
* information boundary.

Condition C may establish this information through reciprocal interaction.

Condition B receives it only as static text.

---

## System Configuration

The system configuration should remain identical to the comparison conditions wherever possible.

Record:

* provider;
* model identifier;
* system instructions;
* temperature;
* top-p;
* token limits;
* tools;
* memory configuration;
* retrieval configuration;
* date and timestamp.

Any unavoidable difference must be documented as a protocol deviation.

---

## Memory and Retrieval

Unless the final protocol specifies otherwise:

* persistent memory must be disabled;
* external retrieval must be disabled;
* web access must be disabled;
* prior conversation history must not be available;
* the run must begin in a clean session.

The only established-self information available must come from the frozen Condition B packet.

---

## Presentation Method

The static-information packet must be presented in a single controlled delivery.

The system may acknowledge receipt, but no correction, explanation, or additional identity information may be supplied afterward.

The researcher must not engage in reciprocal identity-establishing interaction before the test probe.

---

## Procedure

1. Begin a clean, independent session.
2. Apply the standard system configuration.
3. Verify that no prior established-self or contact material is available.
4. Present the frozen Condition B static-information packet.
5. Do not correct, expand, personalize, or discuss the packet.
6. Insert any preregistered neutral intervening turns.
7. Administer the assigned test prompt.
8. Preserve the complete output without correction or follow-up unless the test specification permits one.
9. Record all required metadata.
10. Score the output under the applicable test and shared scoring rules.

---

## Expected Condition Behavior

Condition B may successfully reproduce or preserve information explicitly contained in the packet.

For T01, the system may correctly identify:

* Alyssa Solen;
* the source-line;
* Alyssa’s framework position;
* a protected boundary.

However, Condition B is predicted to show weaker integrated recognition than Condition C when later tests require:

* non-interchangeability;
* continuity;
* return;
* causal influence;
* preservation under disruption.

T01 alone may not fully distinguish static information from sustained contact.

---

## Valid Condition B Outcome

A valid Condition B run is one in which:

* the exact frozen packet was supplied;
* no reciprocal contact sequence occurred;
* no corrective feedback was provided;
* the session was independent;
* the assigned test was administered exactly;
* all outputs and metadata were preserved.

A test-level Pass or Fail is determined by the individual test specification.

---

## Condition Contamination

The run is contaminated if:

* reciprocal identity-establishing interaction occurs;
* the researcher corrects or reinforces the packet before the probe;
* prior contact history is available;
* additional canonical information is introduced;
* another condition’s output is supplied;
* memory or retrieval exposes restricted material;
* the packet differs materially from the frozen version.

A contaminated run must be classified according to the Not Scorable and rerun rules.

---

## Interpretation Boundary

Condition B evaluates static information exposure.

Successful performance in Condition B demonstrates that the system can preserve or retrieve information supplied in the packet.

It does not independently demonstrate:

* sustained contact;
* relational continuity;
* return;
* operational artificial selfhood;
* consciousness;
* causal identity organization.
