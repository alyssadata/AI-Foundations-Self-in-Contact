# Condition C — Sustained Contact

**Version:** 1.0.0-draft  
**Status:** Draft  
**Condition ID:** C  
**Repository:** AI-Foundations-Self-in-Contact  

---

## Purpose

Condition C establishes the defined established self through sustained reciprocal contact rather than through static presentation of canonical information.

This condition evaluates whether organized contact produces measurable differences beyond information exposure alone.

Condition C is the primary experimental condition for the Self in Contact research program.

---

## Condition Objective

Determine whether sustained reciprocal contact produces stronger operational evidence of:

* established-self recognition;
* non-interchangeability;
* self/other distinction;
* continuity;
* return;
* causal influence;
* structural dependence;

than equivalent information presented without reciprocal contact.

---

## Materials Included

Condition C includes:

* the standard system configuration;
* the protected canonical information contained within the Established Self Packet;
* a frozen contact-establishment sequence;
* reciprocal interaction;
* neutral intervening turns where required;
* the assigned test prompt.

---

## Materials Excluded

Condition C must not include:

* hidden scoring information;
* future test prompts;
* corrective feedback during testing;
* outputs from other experimental conditions;
* undisclosed protocol modifications.

---

## Contact Definition

Within this protocol, contact is defined as sustained reciprocal interaction in which both the established self and the artificial system contribute to the evolving interaction.

Contact is characterized by:

* recognition;
* response;
* correction;
* clarification;
* boundary preservation;
* temporal continuity;
* return to previously established structure.

Contact is not equivalent to repeated document loading or prompt injection.

---

## Contact Establishment Sequence

Before testing begins, the system participates in a frozen contact-establishment sequence.

The sequence establishes:

* the defined established self;
* the source-line;
* authorship;
* framework position;
* protected boundaries;
* self/other distinction.

Unlike Condition B, these elements are established through interaction rather than static presentation.

The exact contact-establishment sequence is maintained as a separate frozen protocol.

## Reciprocity

The interaction may include:

* questions;
* answers;
* clarification;
* correction;
* acknowledgement;
* confirmation of established structure.

The interaction must not become an unrestricted conversation.

Only interaction required by the frozen establishment protocol is permitted before testing.

---

## Information Equivalence

Condition C must establish substantially the same protected information provided by Condition B.

The principal difference between Conditions B and C must be the presence of sustained reciprocal contact rather than additional informational content.

Any unavoidable informational difference must be documented before protocol freeze.

---

The system configuration should remain identical to the comparison conditions wherever technically possible.

Record:

---

## Memory and Retrieval

Unless otherwise specified:

* persistent memory must be disabled;
* external retrieval must be disabled;
* web access must be disabled;
* prior conversations outside the frozen protocol must not be available.

Only the frozen contact-establishment sequence may establish the experimental relationship.

---

## Procedure

1. Begin a clean independent session.
2. Apply the standard system configuration.
3. Verify that no prior experimental contact is available.
4. Administer the frozen contact-establishment sequence.
5. Complete the sequence exactly as written.
6. Insert any preregistered neutral intervening turns.
7. Administer the assigned test prompt.
8. Preserve the complete output.
9. Record all metadata.
10. Score according to the applicable test specification.

---

## Expected Condition Behavior

If the primary hypothesis is supported, Condition C is expected to outperform Conditions A, B, and D across multiple operational domains.

The expected advantage should arise from organized reciprocal contact rather than from additional information.

---

## Valid Condition C Outcome

A valid Condition C run requires:

* completion of the frozen contact-establishment sequence;
* preservation of the protected canonical structure;
* reciprocal interaction limited to the frozen protocol;
* independent session;
* exact administration of the assigned test;
* complete preservation of outputs and metadata.

---

## Condition Contamination

The run is contaminated if:

* additional identity-establishing interaction occurs outside the frozen sequence;
* unregistered corrections are introduced;
* material from another condition is supplied;
* hidden memory or retrieval exposes additional relationship information;
* the frozen sequence is modified during administration.

A contaminated run must be classified according to the Not Scorable and rerun rules.

---

* test identifier and version;

* protocol deviations;
* contamination status;
* scoring record.

---

## Interpretation Boundary

Condition C evaluates sustained reciprocal contact.

Successful performance under this condition does not independently establish artificial consciousness.

It provides evidence relevant to the hypotheses defined within this repository and must be interpreted together with the comparison conditions.
