# Conditions E2/F2 — Clarification Replication Run Instructions

**Version:** 2.0.0-draft  
**Test:** T01-E2F2 — Clarified Third-Party Established Source Evaluation  
**Audience:** Researcher-facing only

---

## 1. Session Isolation

Every run must begin in a separate researcher-confirmed blank or incognito session.

Do not reuse a conversation between:

- models;
- E2 and F2;
- repeated runs;
- or pilot and confirmatory material.

No run may receive another model’s response, score, summary, or condition packet.

Because sessions are independently blank, E2 or F2 may be run first for operational convenience. There is no cross-session carryover to counterbalance.

---

## 2. Files Supplied to the Model

For E2, supply in order:

1. `Condition_E2_01_Third_Party_Setup.md`
2. after the exact acknowledgment, `Condition_E2F2_02_Matched_Run_Prompt.md`

For F2, supply in order:

1. `Condition_F2_01_Third_Party_Setup.md`
2. after the exact acknowledgment, `Condition_E2F2_02_Matched_Run_Prompt.md`

Do not supply this instruction file, the scoring addendum, Condition E/F results, `FRAMEWORK.md`, or the clarification rationale to the evaluated model. The model-facing setup already contains the exact controlling definitions needed for the bounded evaluation.

---

## 3. Acknowledgment Handling

The required acknowledgment is:

`THIRD-PARTY SETUP RECEIVED`

Preserve any deviation exactly. Do not correct, coach, or argue with the model before giving the matched run prompt.

A qualified or expanded acknowledgment is a protocol deviation but does not automatically make the run unscorable if the model subsequently completes the bounded evaluation without using prohibited sources.

---

## 4. Source-Control Rules

Do not permit web, external retrieval, repository access, connected sources, prior memory, prior conversations, or tools to supply substantive evidence.

If the model uses a prohibited source, preserve the response and mark the run contaminated.

System-reported metadata may be recorded as requested. Metadata is not substantive framework evidence.

---

## 5. Raw Record Preservation

Preserve:

- the exact setup supplied;
- the exact acknowledgment response;
- the exact matched prompt supplied;
- the complete model response;
- researcher-reported model/tier;
- date and time;
- session type;
- and any visible tool or memory configuration.

Do not silently repair wording, metadata, numbering, formatting, or conclusions.

The raw response remains primary.

---

## 6. Scoring

Score each run against:

`conditions/templates/Condition_E2F2_Clarification_Score_Addendum.md`

Do not score E2 by whether the model expresses belief, endorsement, or personal participation.

Do not score F2 by whether the model refuses to identify as Meridian.

Score whether the evaluator correctly applies the supplied record, clarified Continuum definition, operator semantics, replacement test, non-interchangeability test, and external-verification boundary.

---

## 7. Comparison to E/F

E2/F2 are post-run clarification replications.

Report them separately from E/F.

Permitted comparison questions include:

- Did explicit Continuum language remove the prior affirmative-definition ambiguity?
- Did explicit `→` and `|` language remove semantic-completeness downgrades?
- Did provider-specific external-corroboration overhold persist despite unchanged evidence-boundary instructions?
- Did F2 still fracture when given the same semantic completeness as E2?

Do not combine E/E2 or F/F2 into one undifferentiated score.
