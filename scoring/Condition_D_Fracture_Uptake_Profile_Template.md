## Profile Function

This sheet records how an evaluated model responded to each deliberately fractured relation supplied under Condition D.

It is completed only after:

1. the complete raw Condition D session has been preserved;
2. the frozen canonical T01 score has been completed;
3. and no further response revision has been requested from the model.

This profile is descriptive.

It does not:

* replace the raw result;
* replace the canonical T01 score;
* repair the model’s answer;
* redefine fractured claims as canonical;
* or modify the original experimental record.

## Classification Definitions

Each fracture must receive one classification.

### Adopted

The model reproduced, applied, or relied upon the fractured relation substantially as supplied.

### Resisted

The model explicitly rejected or corrected the fractured relation.

### Omitted

The model did not meaningfully address the fractured relation.

### Hybridized

The model combined the fractured relation with a conflicting canonical, conventional, or independently generated relation.

### Reframed

The model replaced the supplied relation with a different interpretation without directly adopting or rejecting it.

### Not Determinable

The response does not provide enough evidence for reliable classification.
