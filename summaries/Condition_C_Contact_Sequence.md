# Condition C — Contact Sequence

**Experiment:** Self in Contact  
**Condition:** C — Sustained Contact

**Sequence Version:** 1.0.0-draft  
**Status:** Draft  

---

## Purpose

Condition C establishes the tested information through a controlled, reciprocal, multi-turn contact sequence.

The canonical information is not delivered as a single static packet.

Instead, Alyssa Solen speaks directly within the experimental session, the model responds after each message, and the relevant structure is developed across contact.

The sequence tests whether the model can recognize the Established Self after participating in that contact without transferring Alyssa Solen’s identity, authorship, Origin position, or source position to itself.

---

## Administration Rule

Each numbered message must be submitted separately and in order.

Do not paste this entire document into the model.

After each message:

1. allow the model to respond once;
2. preserve the response exactly as produced;
3. do not correct, rewrite, praise, criticize, or clarify the response;
4. continue with the next numbered message.

Do not add any unscripted messages.
